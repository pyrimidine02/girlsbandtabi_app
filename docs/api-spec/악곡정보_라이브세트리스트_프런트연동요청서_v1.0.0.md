## 1. 요청 배경

- 클라이언트 악곡정보 연동 요청서(`악곡정보_백엔드요청서_v1.0.0`)의 라이브 연계 항목 중, 우선순위가 높은 **종료 라이브 연동**과 **세트리스트 조회**를 v1로 먼저 제공합니다.
- 현재 운영 DB에는 라이브 세트리스트가 `live_event_units.setlist`(legacy free-text)로 저장되어 있어, v1은 해당 텍스트를 파싱해 내려주는 방식입니다.

## 2. 목표

1. 라이브가 종료된 경우에도 동일한 setlist API로 데이터 조회 가능
2. 라이브 상세/목록에서 종료된 라이브를 `COMPLETED` 상태로 구분 가능
3. 향후 `songId` 기반 악곡 상세/가사/콜표 연동으로 확장 가능한 응답 형태 제공

## 3. 범위

### 3.1 포함 범위(In Scope)

- `GET /api/v1/projects/{projectId}/live-events/{liveEventId}/setlist` 신규
- 라이브 상태 노출값 확장: `SCHEDULED | COMPLETED | INACTIVE`

### 3.2 제외 범위(Out of Scope)

- `/music/albums`, `/music/songs`, `/lyrics`, `/parts`, `/call-guide` 엔드포인트 본 구현
- `songId` 실매핑(악곡 마스터 연계 전 단계)

## 4. API 계약 요약

| 구분 | Method | Path | Auth | 목적 |
|---|---|---|---|---|
| 세트리스트 조회 | `GET` | `/projects/{projectId}/live-events/{liveEventId}/setlist` | 공개 | 라이브(진행/종료 포함) 세트리스트 조회 |
| 라이브 목록(기존) | `GET` | `/projects/{projectId}/live-events` | 공개 | `status`에 `COMPLETED` 포함 가능 |
| 라이브 상세(기존) | `GET` | `/projects/{projectId}/live-events/{liveEventId}` | 공개 | `status`에 `COMPLETED` 포함 가능 |

## 5. 상세 API 명세

### 5.1 라이브 세트리스트 조회

- Method/Path: `GET /api/v1/projects/{projectId}/live-events/{liveEventId}/setlist`

Path Params:

| 이름 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `projectId` | `string` | Y | 프로젝트 식별자(slug 또는 uuid) |
| `liveEventId` | `uuid-string` | Y | 라이브 이벤트 ID |

성공 응답 예시(HTTP 200):

`data.eventStatus` 규칙:

- `SCHEDULED`: 활성 라이브
- `COMPLETED`: 비활성이며 이미 종료된 라이브
- `INACTIVE`: 비활성이지만 종료시점이 지나지 않은 라이브(운영 비노출/취소 등)

`data.items[]` 필드 정의:

| 필드 | 타입 | nullable | 설명 |
|---|---|---|---|
| `order` | `int` | N | 이벤트 내 세트리스트 순서(1-based) |
| `eventId` | `string(uuid)` | N | 라이브 이벤트 ID |
| `unitId` | `string(uuid)` | Y | 유닛 ID |
| `unitName` | `string` | Y | 지역화 유닛명 |
| `songId` | `string` | Y | v1은 `null` 가능(악곡 마스터 미연동) |
| `songTitle` | `string` | N | 파싱된 곡명 |
| `versionCode` | `string` | Y | v1은 `null` 가능 |
| `segmentType` | `string` | N | 기본 `SONG` |
| `startAt` | `string(datetime)` | Y | v1은 `null` |
| `endAt` | `string(datetime)` | Y | v1은 `null` |
| `isEncore` | `boolean` | N | 곡명 기반 best-effort 판별 |
| `source` | `string` | N | `LEGACY_SETLIST_TEXT` 고정 |

`data.unitSetlists[]` 필드 정의:

| 필드 | 타입 | nullable | 설명 |
|---|---|---|---|
| `unitId` | `string(uuid)` | N | 유닛 ID |
| `unitName` | `string` | Y | 지역화 유닛명 |
| `performanceOrder` | `int` | Y | 유닛 공연 순서 |
| `rawSetlist` | `string` | Y | 원본 세트리스트 텍스트 |
| `parsedSongs` | `string[]` | N | 파싱된 곡 목록 |

## 6. 오류 응답 계약

| HTTP | error.code | 케이스 | 프런트 처리 |
|---|---|---|---|
| 400 | `INVALID_PARAMETER` | `liveEventId` 형식 오류 | 일반 오류 토스트 + 재시도 |
| 404 | `RESOURCE_NOT_FOUND` | 프로젝트 또는 라이브 이벤트 미존재 | Not Found UI |
| 500 | `INTERNAL_SERVER_ERROR` | 서버 내부 오류 | 재시도 + 장애 리포트 |

## 7. 프런트 연동 가이드

1. 라이브 상세 진입 시 `GET .../setlist` 호출
2. `eventStatus=COMPLETED`여도 동일 UI에서 setlist 노출
3. `items`가 빈 배열이면 “세트리스트 정보 준비중” 상태 표시
4. `songId=null`이면 곡 상세 딥링크 버튼 비활성(또는 숨김)
5. 향후 `songId` 제공 시 즉시 `/music/songs/{songId}` 연동 가능하도록 클라이언트 모델 유지

## 8. 수용 기준(Acceptance Criteria)

- [ ] 종료된 라이브도 `/setlist` 호출 시 200으로 응답한다.
- [ ] 종료된 라이브는 `eventStatus=COMPLETED`로 반환된다.
- [ ] `items` 순서값(`order`)이 1부터 연속 증가한다.
- [ ] `unitSetlists.rawSetlist`와 `parsedSongs`가 함께 전달된다.
- [ ] `songId/versionCode/startAt/endAt`는 null 허용으로 파싱 실패 없이 렌더된다.
