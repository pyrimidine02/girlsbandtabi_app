## 1. 적용 범위

- 앨범 목록/상세
- 곡 목록/상세
- 가사/파트/콜표
- 버전/크레딧/난이도/미디어/가용성
- 라이브 세트리스트 및 라이브-곡 통합 컨텍스트

| Method | Path | 설명 |

| `GET` | `/projects/{projectId}/music/songs` | 곡 커서 목록 |

| `GET` | `/projects/{projectId}/music/songs/{songId}/versions` | 버전 목록 |

## 3. 목록 API 커서 계약

`/music/albums`, `/music/songs`

- 요청: `cursor`, `size`(기본 20)
- 응답 data:
  - `items[]`
  - `nextCursor`
  - `hasNext`

예시:

```json
{
  "success": true,
  "statusCode": 200,
  "data": {
    "items": [],
    "nextCursor": null,
    "hasNext": false
  }
}
```

## 4. 핵심 응답 필드

### 4.1 가사
- `lineId`, `order`, `startMs`, `endMs`, `section`
- `textOriginal`, `textRomanized`, `textTranslated`

### 4.2 파트
- `segmentId`, `startMs`, `endMs`
- `memberId`, `memberName`, `unitId`, `unitName`
- `partType`, `lyricLineId`

### 4.3 콜표
- `cueId`, `startMs`, `endMs`, `cueType`
- `cueText`, `intensity`, `target`, `note`

### 4.4 라이브 세트리스트
- `eventStatus`: `SCHEDULED | COMPLETED | INACTIVE`
- `items[]`:
  - `order`, `eventId`, `songId`, `songTitle`, `versionCode`
  - `segmentType`, `startAt`, `endAt`, `isEncore`, `source`

## 5. 파라미터 규칙

- `projectId`: slug 또는 UUID
- `songId`, `albumId`, `liveEventId`: UUID
- `version`: `FULL | TV_SIZE | LIVE_EDIT | INSTRUMENTAL`
- `lang`: `ko | ja | en` (기타 요청은 서버 폴백)
- `country`: ISO-3166 alpha-2 (예: `KR`, `JP`)

## 6. 에러 처리

| HTTP | 케이스 |
|---|---|
| 400 | 잘못된 UUID/파라미터 |
| 404 | 프로젝트/앨범/곡/라이브 미존재 |
| 500 | 서버 내부 오류 |

## 7. 프런트 적용 체크리스트

1. 곡/앨범 목록은 cursor 기반 상태(`items`, `nextCursor`, `hasNext`)로 구현
2. 가사 화면에서 `includeRomanized`, `includeTranslated` 토글 반영
3. 라이브 상세에서 `setlist` 호출 시 `eventStatus=COMPLETED`도 동일 렌더
4. `songId`가 null인 세트리스트 항목은 곡 상세 딥링크 비활성
5. 라이브 컨텍스트는 `eventId` 필수로 호출
