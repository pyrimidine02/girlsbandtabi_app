**Girls Band Tabi - 앨범/가사/파트/콜표 악곡 정보 기능 연동 요청**

- 작성일: 2026-03-12
- 버전: v1.0.0
- 상태: 🟡 백엔드 검토 요청
- 작성: girlsbandtabi Flutter 클라이언트팀

## 1) 목적
- 앱 내 악곡 정보를 프로젝트별 단순 목록에서 실제 감상/입덕/현장(라이브) 활용 가능한 구조로 확장합니다.
- 특히 아래 4개 핵심 기능을 안정적으로 제공하기 위한 서버 계약이 필요합니다.
  1. 앨범 정보
  2. 노래별 가사
  3. 가사 구간별 멤버 파트
  4. 노래별 콜 타이밍(콜표)

## 2) 핵심 기능 범위

### 2.1 필수(이번 요청)
1. 앨범 목록/상세
2. 앨범-트랙(곡) 관계
3. 곡 상세 메타데이터
4. 다국어 가사(원문/번역/로마자)
5. 가사 타임라인 + 파트(멤버/유닛)
6. 콜표 타임라인(초 단위/밀리초 단위)

### 2.2 확장 요구(상세 계약 필요)
1. 버전별 악곡 정보
  - `FULL`, `TV_SIZE`, `LIVE_EDIT`, `INSTRUMENTAL` 고정 enum
  - 버전별 `durationMs`, `isDefault`, `audioFeatures(bpm/key)` 분리 제공
  - 버전별 가사/파트/콜표 데이터 매칭 가능해야 함
2. 라이브 세트리스트 연동
  - 라이브 상세에서 곡 카드 탭 시 곡 상세/가사/콜표로 이동
  - 세트리스트 항목에 `order`, `songId`, `versionCode`, `startAt` 제공
  - 앵콜/MC 구간 포함 여부 플래그 필요
3. 악곡 크레딧
  - 작사/작곡/편곡/보컬 참여 멤버를 역할 단위로 제공
  - 1곡 내 다중 참여자(공동 작곡/공동 작사) 지원 필요
4. 난이도/콜 강도 지표
  - 난이도: `BEGINNER | INTERMEDIATE | ADVANCED`
  - 콜 강도: `1..5` 정수
  - 난이도 산정 근거(tempo, cueDensity) 메타 제공 권장
5. 오디오 프리뷰 링크
  - 15~30초 미리듣기 URL
  - 외부 스트리밍(Spotify/Apple Music/YouTube Music) 링크 제공
  - 국가별 재생 제한 시 fallback 링크 정책 필요
6. 콘텐츠 가용성 메타
  - 국가/권리 제한(`allowedCountries`, `blockedCountries`)
  - 공개 시작/종료 시각(`availableFrom`, `availableUntil`)
  - 현재 노출 가능 여부(`isAvailableNow`) 계산값 포함

## 3) API 요청안 (클라이언트 조회 기준)

> 공통 응답 래퍼: 기존 `ApiResponse` 규격 사용  
> 시간 단위: `ISO-8601(UTC)` / 타임라인은 `ms` 기준  
> 인증: 조회는 `선택(Auth optional)` 권장, 개인화(즐겨찾기/최근 본 곡)는 `필수`

### 3.1 앨범
- `GET /api/v1/projects/{projectCode}/music/albums`
  - Query: `cursor?`, `size=20`, `q?`, `releaseFrom?`, `releaseTo?`
  - Response: `items[], nextCursor, hasNext`
- `GET /api/v1/projects/{projectCode}/music/albums/{albumId}`
  - Response: 앨범 상세 + 트랙 요약

### 3.2 곡
- `GET /api/v1/projects/{projectCode}/music/songs`
  - Query: `cursor?`, `size=20`, `q?`, `unitId?`, `albumId?`, `sort?`
  - Response: `items[], nextCursor, hasNext`
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}`
  - Response: 곡 상세(버전 목록 포함)

### 3.3 가사
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/lyrics`
  - Query: `version=FULL`, `lang=ja`, `includeRomanized=true`, `includeTranslated=true`
  - Response: 가사 라인 배열(타임라인 포함)

### 3.4 파트(멤버/유닛)
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/parts`
  - Query: `version=FULL`
  - Response: 파트 구간 배열(`startMs`, `endMs`, `memberId`, `memberName`, `partType`)

### 3.5 콜표
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/call-guide`
  - Query: `version=LIVE_EDIT`, `lang=ko`
  - Response: 콜 큐 배열(`startMs`, `endMs`, `cueType`, `cueText`, `intensity`)

### 3.6 라이브 세트리스트(추가 제안)
- `GET /api/v1/projects/{projectCode}/live-events/{eventId}/setlist`
  - Response: 공연 곡 순서 + 곡 ID 연결

### 3.7 버전별 악곡 정보(상세)
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/versions`
  - Response: 사용 가능한 버전 목록 + 기본 버전
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/versions/{versionCode}`
  - Response: 해당 버전의 duration/bpm/key/timeSignature/arrangementNote

### 3.8 악곡 크레딧(상세)
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/credits`
  - Query: `includeMembers=true`
  - Response: 작사/작곡/편곡/보컬 참여자 목록(역할별 배열)

### 3.9 난이도/콜 강도 지표
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/difficulty`
  - Query: `version=FULL`
  - Response: `difficultyLevel`, `callIntensity`, `metrics(cueDensity, vocalRange, tempoScore)`

### 3.10 오디오 프리뷰/외부 링크
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/media-links`
  - Response: preview clip + provider 링크 + 지역 제한 정보

### 3.11 콘텐츠 가용성 메타
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/availability`
  - Query: `country=KR`
  - Response: `isAvailableNow`, `availableFrom`, `availableUntil`, 지역 제한 메타

### 3.12 라이브 상세 즉시 진입용 통합 응답(옵션)
- `GET /api/v1/projects/{projectCode}/music/songs/{songId}/live-context`
  - Query: `eventId`, `version=LIVE_EDIT`, `lang=ko`
  - Response: `song + lyrics + parts + callGuide + setlistContext` 묶음(1회 요청)

## 4) 데이터 계약(요약)

### 4.1 AlbumSummary
- `id`, `projectId`, `title`, `type(SINGLE|EP|ALBUM|LIVE_ALBUM)`, `coverUrl`
- `releaseDate`, `trackCount`, `label`, `catalogNo`

### 4.2 SongSummary / SongDetail
- `id`, `projectId`, `title`, `titleJa`, `titleEn`, `durationMs`, `bpm`
- `primaryUnitId`, `primaryUnitName`
- `albumId`, `trackNo`, `isTitleTrack`
- `versions[]` (`versionCode`, `durationMs`, `isDefault`)
- `credits` (작사/작곡/편곡)
- `previewUrl`, `streamingLinks[]`

### 4.3 LyricLine
- `lineId`, `order`, `startMs`, `endMs`
- `textOriginal`, `textRomanized?`, `textTranslated?`
- `section(VERSE|PRE_CHORUS|CHORUS|BRIDGE|OUTRO)`

### 4.4 PartSegment
- `segmentId`, `startMs`, `endMs`
- `memberId`, `memberName`, `unitId`, `unitName`
- `partType(SOLO|DUET|UNISON|HARMONY|RAP|SHOUT)`
- `lyricLineId?`

### 4.5 CallCue
- `cueId`, `startMs`, `endMs`
- `cueType(CALL|CLAP|CHANT|JUMP|WOTAGEI|SILENCE)`
- `cueText`, `intensity(1..5)`, `target(L|R|CENTER|ALL)`
- `note?` (주의사항/현장 팁)

### 4.6 SongVersionInfo
- `versionCode(FULL|TV_SIZE|LIVE_EDIT|INSTRUMENTAL)`
- `durationMs`, `bpm`, `key`, `timeSignature`, `isDefault`
- `arrangementNote?`

### 4.7 SongCredit
- `role(LYRICIST|COMPOSER|ARRANGER|VOCAL)`
- `contributors[]`:
  - `id?` (멤버 매핑 가능 시)
  - `name`
  - `type(MEMBER|EXTERNAL)`

### 4.8 DifficultyProfile
- `difficultyLevel(BEGINNER|INTERMEDIATE|ADVANCED)`
- `callIntensity(1..5)`
- `metrics`:
  - `cueDensityPerMin`
  - `vocalRangeScore(1..5)`
  - `tempoScore(1..5)`

### 4.9 SongMediaLink
- `preview`:
  - `url`, `durationSec`, `waveformUrl?`
- `streamingLinks[]`:
  - `provider(SPOTIFY|APPLE_MUSIC|YOUTUBE_MUSIC|LINE_MUSIC)`
  - `url`
  - `regionAvailability?`

### 4.10 AvailabilityMeta
- `isAvailableNow`
- `availableFrom`, `availableUntil`
- `allowedCountries[]`, `blockedCountries[]`
- `rightsPolicy(COPYRIGHT_BLOCKED|PRE_RELEASE|EXPIRED|OK)`

### 4.11 LiveSetlistItem
- `order`, `eventId`
- `songId`, `songTitle`
- `versionCode`
- `segmentType(SONG|MC|ENCORE)`
- `startAt?`, `endAt?`
- `isEncore`

## 6) 백엔드 구현 요청사항 (핵심)
1. **버전/언어 파라미터 표준화**
  - `version`, `lang` 파라미터를 모든 악곡 하위 API에서 공통 사용
2. **타임라인 정합성 보장**
  - `lyrics`, `parts`, `callGuide`의 `startMs/endMs`가 같은 버전 기준으로 정렬
3. **계약 안정성**
  - 존재하지 않는 곡/버전/언어는 명시적 에러코드 반환
4. **성능**
  - 가사/파트/콜표는 최초 응답 300ms~700ms 목표
  - ETag/Last-Modified 기반 조건부 조회 지원
5. **운영 입력 경로**
  - 대량 데이터 업로드용 Admin/CMS 입력 API 또는 배치 포맷(예: CSV/JSON) 제공
6. **확장 6항목 계약화**
  - 버전/세트리스트/크레딧/난이도/미디어/가용성 항목을 별도 엔드포인트 또는 곡 상세 확장 필드로 제공
7. **라이브 연동 우선**
  - 라이브 상세에서 곡 상세로 이동 시 1~2회 요청으로 필요한 데이터(가사/콜표 포함) 확보 가능해야 함
8. **권리/지역 정책 명시**
  - 재생 불가 국가/기간의 처리 규칙(HTTP status + 응답 body)을 고정 계약으로 문서화

## 7) 추가 제한사항 (전부 반영)

### 7.1 Query/입력 파라미터 제한
- `size`: `1..100` (default `20`)
- `cursor`: 최대 길이 `256`
- `q`: trim 후 `1..80`자(빈 문자열 불가)
- `lang`: `ko | en | ja`
- `version`: `FULL | TV_SIZE | LIVE_EDIT | INSTRUMENTAL`
- `sort`: 서버 허용값 외 요청 시 `400 INVALID_REQUEST`
- `includeRomanized/includeTranslated`:
  - `lyrics` API에서만 허용
  - 기타 엔드포인트에서 전달 시 무시하지 말고 `400` 반환 권장

### 7.2 타임라인 무결성 제한
- 모든 타임라인 필드(`startMs`, `endMs`)는 `0 <= startMs < endMs`
- `endMs`는 해당 곡 버전 `durationMs`를 초과할 수 없음
- `lyrics.order`는 `1`부터 연속 증가(중복/누락 불가)
- `parts.lyricLineId`가 존재하면 반드시 실제 `lyrics.lineId`를 참조
- `parts` 겹침 규칙:
  - `DUET/UNISON/HARMONY`는 겹침 허용
  - 그 외 타입은 동일 멤버 기준 겹침 불가
- `callGuide`는 `startMs` 오름차순 정렬 보장

### 7.3 필드 길이/개수 제한
- 가사 라인 수: 최대 `600`
- 파트 세그먼트 수: 최대 `1200`
- 콜 큐 수: 최대 `1500`
- `textOriginal/textRomanized/textTranslated`: 각 라인 최대 `140`자
- `cueText`: 최대 `60`자
- `note`: 최대 `120`자
- 응답 용량 권장 상한: `~1MB`
  - 상한 초과 예상 시 `fromMs`/`toMs` 범위 조회 파라미터 지원 권장

### 7.4 캐시/리비전 제한
- `albums/{id}`, `songs/{id}`, `lyrics`, `parts`, `call-guide` 응답에 `ETag` 제공
- `If-None-Match` 조건부 조회 시 `304` 지원
- 응답에 `revision`(정수) + `updatedAt` 포함
  - 동일 곡/동일 버전 데이터 변경 시 `revision` 증가

### 7.5 레이트 리밋 제한
- 일반 조회 API: 사용자 기준 `120 req/min` 권장
- 검색형 API(`q` 포함): 사용자 기준 `30 req/min` 권장
- 초과 시 `429` + `Retry-After`(초) 반환

### 7.6 상태코드 매핑 제한(권장)
- `400`: 파라미터 형식 오류/필수값 누락
- `401`: 인증 필요 엔드포인트 미인증
- `403`: 국가/권리 제한으로 비공개
- `404`: 앨범/곡/가사/파트/콜표 리소스 없음
- `409`: 운영 입력(CMS) 리비전 충돌
- `413`: 운영 입력 payload 초과
- `422`: 버전/언어 미지원, 타임라인 무결성 실패
- `429`: 호출 빈도 제한

### 7.7 확장 항목 제약
- `versions`:
  - 곡당 최소 1개, 최대 8개
  - `isDefault=true`는 정확히 1개만 허용
- `preview.durationSec`: `15..30`
- `difficulty.callIntensity`: `1..5` (정수만)
- `difficultyLevel`: `BEGINNER|INTERMEDIATE|ADVANCED` 외 값 금지
- `streamingLinks`:
  - provider 중복 금지
  - URL은 `https`만 허용
- `availability`:
  - `allowedCountries`/`blockedCountries`는 ISO 3166-1 alpha-2 코드만 허용
  - 두 배열 동시 존재 시 동일 코드를 중복 포함할 수 없음
- `setlist.items`:
  - `order`는 1부터 연속
  - `segmentType=SONG`인 경우 `songId`, `versionCode` 필수
  - `segmentType=MC`인 경우 `songId`, `versionCode`는 null 고정
