### 2.2 Live 연동 (기존 반영분 포함)

| Method | Path | 설명 |
|---|---|---|
| `GET` | `/projects/{projectId}/live-events/{liveEventId}/setlist` | 라이브 세트리스트 조회 (진행/종료 라이브 모두) |

추가 반영 규칙:
- `GET /projects/{projectId}/live-events`
- `GET /projects/{projectId}/live-events/{liveEventId}`

위 2개 기존 API의 `status`는 종료된 비활성 이벤트에서 `COMPLETED`를 반환합니다.

## 3. 프런트에서 꼭 반영할 계약 포인트

1. 목록 API(`albums`, `songs`)는 cursor 계약 사용
- `data.items[]`, `data.nextCursor`, `data.hasNext`

2. 버전 코드 enum 고정
- `FULL | TV_SIZE | LIVE_EDIT | INSTRUMENTAL`

3. 종료 라이브 처리
- 세트리스트 API에서 `eventStatus=COMPLETED`도 정상 렌더

4. 세트리스트 데이터 소스 정책
- 서버는 `live_event_setlist_items`(구조화 데이터) 우선
- 없으면 `live_event_units.setlist`(legacy free-text) fallback
- 따라서 `songId`는 null 가능 (fallback 데이터)

5. 다국어
- `lyrics/call-guide/live-context`에서 `lang` 파라미터 수용
- 미지정 시 서버 locale fallback 적용

6. 가용성
- `availability`는 `country`(예: `KR`, `JP`)에 따라 `isAvailableNow` 계산값 제공

## 4. 권장 화면 매핑

- 앨범 탭: `/music/albums`, `/music/albums/{albumId}`
- 곡 리스트/상세: `/music/songs`, `/music/songs/{songId}`
- 가사 탭: `/lyrics` (`includeRomanized/includeTranslated` 토글 연동)
- 파트 탭: `/parts`
- 콜표 탭: `/call-guide`
- 라이브 상세 곡 진입: `/live-events/{liveEventId}/setlist` -> `songId` 존재 시 `songs/{songId}` 이동
- 원샷 화면: `/live-context?eventId=...`

## 5. 에러 처리 기준

| HTTP | 의미 | 프런트 처리 |
|---|---|---|
| 400 | 파라미터/UUID 오류 | 사용자 일반 오류 + 재시도 |
| 404 | 프로젝트/앨범/곡/라이브 미존재 | Not Found 상태 화면 |
| 500 | 서버 오류 | 재시도 + 장애 리포트 |
