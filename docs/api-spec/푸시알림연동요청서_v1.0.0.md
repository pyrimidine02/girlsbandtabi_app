## 1. 배경
- 사용자 요구: 앱에서 알림이 실제로 울리고(배너/사운드), 앱이 백그라운드/종료 상태여도 도착해야 함.
- 현재 앱은 SSE + polling fallback 기반으로 최신 알림을 가져오며, foreground에서는 로컬 알림 표시까지 구현됨.
- 백그라운드/종료 상태 알림은 APNs(ios)/FCM(android) 서버 발송 파이프라인 없이는 불가능함.

## 2. 현재 클라이언트 구현 상태
- SSE endpoint 구독:
  - `/api/v1/notifications/stream`
- 신규 미읽음 알림 감지 시 local notification 표시:
  - `flutter_local_notifications`
- 사용자 설정 `pushEnabled`와 로컬 표시 정책 동기화 완료.
- 아직 미구현:
  - 디바이스 토큰 서버 등록/해제 API 연동
  - 백엔드 APNs/FCM 실제 발송

## 3. 백엔드 요청 사항 (필수)

### 3.1 디바이스 등록 API
- `POST /api/v1/notifications/devices`
- 목적: 사용자 디바이스의 푸시 토큰 등록
- 요청 예시:
```json
{
  "platform": "IOS",
  "provider": "APNS",
  "deviceId": "ios-vendor-or-generated-id",
  "pushToken": "<apns_or_fcm_token>",
  "appVersion": "0.0.3+2026030601",
  "locale": "ko-KR",
  "timezone": "Asia/Seoul"
}
```

### 3.2 토큰 갱신 API
- `PATCH /api/v1/notifications/devices/{deviceId}/token`
- 목적: APNs/FCM 토큰 회전 시 최신 토큰 반영

### 3.3 디바이스 해제 API
- `DELETE /api/v1/notifications/devices/{deviceId}`
- 목적: 로그아웃/탈퇴/앱 삭제 시 발송 대상 제외

### 3.4 발송 API (내부/관리자 트리거 또는 이벤트 파이프라인)
- 커뮤니티/시스템 이벤트 발생 시 사용자별 푸시 발송 큐 적재
- 최소 이벤트:
  - 댓글/답글 도착
  - 게시글 좋아요/북마크 등 사용자 설정상 허용된 이벤트
  - 운영 공지/시스템 알림

## 4. 이벤트/페이로드 계약

### 4.1 푸시 payload 표준 키
- `notificationId` (필수): 앱 내 상세 조회 키
- `type` (필수): 예) `COMMENT_CREATED`, `SYSTEM_NOTICE`
- `title`, `body` (필수)
- `projectCode` (선택)
- `entityId` (선택): postId/commentId 등
- `deeplink` (선택): 앱 라우팅 경로

### 4.2 중복 방지/멱등성
- 동일 `notificationId`는 중복 발송/중복 저장 방지
- 클라이언트는 `notificationId` 기준으로 화면 갱신/중복 표시 방지

### 4.3 SSE와 Push 정합성
- SSE 이벤트와 Push payload가 동일 `notificationId`를 공유하도록 통일
- 클라이언트가 foreground에서는 SSE+로컬 알림, background에서는 OS 푸시를 동일 데이터로 처리 가능해야 함

## 5. 운영 정책 요청
- Rate limit 정책 문서화:
  - 사용자별/디바이스별 분당 발송 한도
- 실패 재시도 정책:
  - APNs/FCM transient 실패 백오프
- 토큰 무효화 정책:
  - Invalid token 응답 시 자동 비활성화
- 관측 지표:
  - enqueue 성공률, provider 성공률, open/deeplink 전환률

## 6. 단계별 적용 계획
1. **Phase A (백엔드 우선)**: 디바이스 등록/해제 API + APNs/FCM 발송 파이프라인 구축
2. **Phase B (클라이언트)**: `firebase_messaging` 연동, 토큰 등록 API 호출, 푸시 클릭 딥링크 처리
3. **Phase C (검증)**: foreground/background/terminated 시나리오 통합 QA
