# 광고 슬롯 하이브리드 연동 요청서 v1.0.0

## 1) 목적
- 앱의 인라인 광고 슬롯에서 다음을 동시에 운영하기 위함:
  - 외부 광고 네트워크(AdMob Native)
  - 하우스 캠페인(개인 프로모션/서비스 유입)
- 강제 전환형(interstitial/timed redirect) 광고는 사용하지 않음.

## 2) 요청 API

### A. 슬롯 결정 조회
- `GET /api/v1/ads/decision`
- Query
  - `slot` (required): `home_primary` | `board_feed`
  - `ordinal` (optional): 슬롯 순번(0-based)
  - `projectKey` (optional): 현재 프로젝트 코드/ID

#### Response (success)
```json
{
  "success": true,
  "statusCode": 200,
  "data": {
    "decisionId": "dec_20260306_001",
    "campaignId": "cmp_202603_house_03",
    "slot": "board_feed",
    "deliveryType": "network",
    "network": "admob",
    "adUnitId": "ca-app-pub-xxxx/yyyy",
    "badgeLabel": "광고",
    "sponsorLabel": "GirlsBandTabi 추천",
    "title": "프로모션 제목",
    "description": "프로모션 설명",
    "ctaLabel": "자세히 보기",
    "targetPath": "/places",
    "targetUrl": null
  },
  "error": null
}
```

#### deliveryType 규칙
- `house`: 하우스 캠페인 카드 렌더
- `network`: 네트워크 광고 렌더(예: AdMob)
- `none`: 슬롯 숨김

### B. 광고 이벤트 수집
- `POST /api/v1/ads/events`
- Body
```json
{
  "eventType": "impression",
  "slot": "board_feed",
  "ordinal": 0,
  "projectKey": "girls-band-cry",
  "decisionId": "dec_20260306_001",
  "campaignId": "cmp_202603_house_03"
}
```

#### eventType
- `impression`
- `click`

## 3) 서버 측 권장 정책
- 빈도 제한:
  - 사용자 기준 슬롯별 노출 제한(예: `N회/일`)
- 세그먼트:
  - 프로젝트/권한/국가 기준 캠페인 노출 제어
- 폴백:
  - network no-fill 시 house로 자동 전환 가능하도록 decision 제공

## 4) 클라이언트 현재 구현 상태
- 클라이언트는 위 두 엔드포인트를 호출할 준비 완료.
- API 미구현 시:
  - 홈: 하우스 캠페인 폴백
  - 게시판: 네트워크 광고 시도 후 하우스 폴백

## 5) 확인 필요 항목
- `projectKey`를 slug/UUID 중 무엇으로 고정할지
- 이벤트 적재 지연 허용시간(SLA)
- `deliveryType=network` 시 `adUnitId` 미포함일 때 정책(=none 또는 house)
