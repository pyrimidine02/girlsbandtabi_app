## 1. 목적
- 프런트에서 적용한 법률 컴플라이언스 UX(동의 수집/정책 고지)를 서버 감사 가능 상태로 확정하기 위한 API 계약 정리.

## 2. 요청 계약

### 2.1 회원가입 동의 필드 확장
- 대상: `POST /api/v1/auth/register`
- 요청 예시:
```json
{
  "username": "user@example.com",
  "password": "*****",
  "nickname": "닉네임",
  "consents": [
    {
      "type": "TERMS_OF_SERVICE",
      "version": "v2026.03.06",
      "agreed": true,
      "agreedAt": "2026-03-06T12:30:00+09:00"
    },
    {
      "type": "PRIVACY_POLICY",
      "version": "v2026.03.06",
      "agreed": true,
      "agreedAt": "2026-03-06T12:30:00+09:00"
    },
    {
      "type": "AGE_OVER_14",
      "version": "v2026.03.06",
      "agreed": true,
      "agreedAt": "2026-03-06T12:30:00+09:00"
    }
  ]
}
```

### 2.2 동의 이력 조회
- 대상: `GET /api/v1/users/me/consents`
- 필요 응답:
  - 동의 항목(`type`)
  - 최신 동의 여부(`agreed`)
  - 동의 시각(`agreedAt`)
  - 정책 버전(`version`)
  - 효력일(`effectiveAt`)

### 2.3 개인정보 설정(자동번역 옵트아웃 포함)
- 대상: `PATCH /api/v1/users/me/privacy-settings`
- 필요 필드:
  - `allowAutoTranslation` (bool)
  - `allowLocationVerification` (bool, optional)

### 2.4 권리행사 API
- 대상:
  - `DELETE /api/v1/users/me` (셀프 탈퇴)
  - `POST /api/v1/users/me/privacy-requests` (열람/처리정지 요청, 선택)

## 3. 정책 메타데이터 API 요청
- 대상: `GET /api/v1/policies/metadata`
- 목적: 프런트 상수 하드코딩 제거, 최신 버전/효력일/URL 동기화
- 필요 필드:
  - `type`, `version`, `effectiveAt`, `url`, `requiresReconsent`
