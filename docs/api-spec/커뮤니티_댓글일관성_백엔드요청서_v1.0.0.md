# 커뮤니티 댓글/답글 일관성 백엔드 요청서 v1.0.0

## 목적
- 모바일 앱에서 댓글/답글 생성/삭제 직후 카운트/활동 목록 일관성을 높이고,
  프로젝트 불일치로 인한 재시도 비용을 줄이기 위한 서버 계약 보강 요청.

## 현상 요약
- 추천/팔로잉 등 전역 피드에서 게시글의 실제 프로젝트와 현재 선택 프로젝트가
  다를 수 있어, 프로젝트 스코프 endpoint 호출 시 실패 가능성이 존재함.
- 댓글 삭제(특히 스레드 포함 삭제) 이후 클라이언트가 정확한 변경량을 알기 어려워
  재조회가 필수이며, UX 지연/추가 트래픽이 발생함.

## 요청 사항
1. 게시글/댓글 응답에 `projectCode` 포함
   - 대상:
     - `PostSummaryResponse`
     - `PostResponse`
     - `CommentResponse`
   - 기대 효과:
     - 클라이언트의 project id→code 변환 및 다중 재시도 제거
     - 전역 피드 진입 후 댓글/답글 API 호출 안정성 향상

2. 게시글 단건 글로벌 조회 endpoint 제공(선택)
   - 예시:
     - `GET /api/v1/community/posts/{postId}`
     - `GET /api/v1/community/posts/{postId}/comments`
   - 기대 효과:
     - 프로젝트 코드 의존 없이 detail/comment 조회 가능
     - 클라이언트 라우팅 단순화

3. 댓글 삭제 응답에 요약 메타 포함(선택)
   - 예시:
     - `deletedCount` (스레드 포함 삭제 개수)
     - `postCommentCount` (삭제 후 게시글 총 댓글 수)
   - 기대 효과:
     - 클라이언트가 즉시 정확한 카운트를 반영 가능
     - 불필요한 강제 재조회 감소

4. 답글 생성 실패 코드 세분화
   - 현재 앱에서 필요한 분기:
     - `COMMENT_THREAD_DEPTH_EXCEEDED`
     - `CONVERSATION_REPLY_NOT_ALLOWED`
     - `BLOCKED_USER_INTERACTION`
     - `POST_NOT_FOUND`
   - 기대 효과:
     - 사용자 안내 문구 정확도 향상
     - 재시도 가능/불가 상황 구분

5. 활동 탭 일관성 이벤트 보장
   - 댓글/게시글 생성·수정·삭제 시 SSE(또는 outbox)로 최소 이벤트 전송:
     - `projectCode`, `postId`, `authorId`, `eventType`, `updatedAt`
   - 기대 효과:
     - 사용자 프로필 `작성한 글/작성한 댓글` 탭의 실시간 갱신 신뢰도 향상

## 우선순위
- P1: `projectCode` 응답 포함
- P1: 답글 실패 코드 세분화
- P2: 삭제 응답 메타(`deletedCount`, `postCommentCount`)
- P3: 글로벌 단건 endpoint

## 모바일측 임시 대응(현재 적용)
- 프로젝트 코드 후보 재시도(404 계열) 로직 적용
- 댓글 생성/삭제 후 강제 재조회(`forceRefresh`) 적용
- mutation 시 post list/detail/author activity 캐시 prefix 무효화 적용
