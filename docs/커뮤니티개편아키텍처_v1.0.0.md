## 1. 개편 목표
- 텍스트 중심 커뮤니티 가독성 강화
- 팔로우/차단/신고 흐름을 서버 계약 기반으로 일원화
- 모바일에서 빠른 스캔(제목/본문 요약/메타/액션) 우선
- 과한 장식 제거, 상태/행동 중심 인터랙션 강화

## 2. 참고 패턴 요약
- X(Twitter): 타임라인형 카드, 액션행 고정 리듬
- Everytime/DCInside/Arca Live: 텍스트 우선, 정보 밀도 높은 목록
- Blind/Toss Community: 신뢰 정보(상태/관계)와 명확한 액션 CTA

## 3. 정보 구조(IA) 개편안
- 커뮤니티 홈
  - 피드 모드: `최신 / 트렌딩 / 구독`
  - 빠른 작성 CTA
- 게시글 상세
  - 본문/미디어
  - 액션 바(좋아요/북마크/댓글)
  - 댓글 스레드
- 사용자 프로필
  - 아이덴티티(아바타/닉네임/소개)
  - 관계 액션(팔로우/차단)
  - 연결 목록 진입(팔로워/팔로잉)
  - 활동 탭(글/댓글)

## 4. Flutter 아키텍처 개편안
- Feature 경계: `lib/features/feed/**` 유지
- 계층 역할
  - `presentation`: 페이지/위젯/라우팅
  - `application`: Riverpod controller/provider
  - `domain`: entity/repository contract
  - `data`: dto/remote datasource/repository impl
- 연결(팔로우) 도메인 모델
  - `UserFollowStatus`: 단일 사용자 관계 + 카운트
  - `UserFollowSummary`: 팔로워/팔로잉 리스트 항목
- 라우팅 확장
  - `/users/:userId/followers`
  - `/users/:userId/following`

## 5. UI/UX 컴포넌트 규칙
- 목록 카드
  - 제목/본문 요약 우선, 메타는 2차 정보
  - 카드 높이 최소화(세로 스캔 최적화)
- 프로필 헤더
  - 장식성 뱃지 최소화, 관계/행동 중심 배치
  - 팔로워/팔로잉은 탭 진입 타일로 명확히 제공
- 입력/탐색
  - 검색 필드 상단 고정(팔로워/팔로잉 목록)
  - Pull-to-refresh 기본 지원

## 6. 단계별 롤아웃 계획
1. **Phase 1 (완료)**: 프로필/연결 목록/팔로우 API 통합
2. **Phase 2 (완료)**: 게시판 컨텍스트 인트로 + 글 상세 작성자 관계 액션 통합
3. **Phase 3 (완료)**: 신고 입력 UI 공용화 + 게시판 포스트 단위 신고/차단 액션 통합
4. **Phase 4 (진행 예정)**: 댓글/답글 레벨 신뢰·모더레이션 UX 통합 및 성능 최적화
