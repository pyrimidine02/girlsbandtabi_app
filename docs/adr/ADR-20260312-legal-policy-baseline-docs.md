# ADR-20260312-legal-policy-baseline-docs

- Date: 2026-03-12
- Status: Accepted

## Context

앱에서 이용약관/개인정보 처리방침/위치정보 이용약관 링크를 노출하고 있으나,
리포지토리에 게시 가능한 최신 법률 문서 마크다운 원본이 정리되어 있지 않았습니다.

운영 정책상, 앱 배포 전에 문서 원본을 버전 관리하고
정책 URL 게시 시 원문과의 정합성을 유지해야 합니다.

## Decision

다음 3개 문서를 `docs/legal/`에 신규 생성하고,
버전을 `v2026.03.12`로 고정해 기준본으로 관리합니다.

1. `docs/legal/이용약관_v2026.03.12.md`
2. `docs/legal/개인정보처리방침_v2026.03.12.md`
3. `docs/legal/위치정보이용약관_v2026.03.12.md`

문서에는 게시 전 반드시 채워야 하는 항목(상호, 대표자, 사업자정보,
개인정보보호책임자/위치정보관리책임자 연락처 등)을 명시적 플레이스홀더로 표시합니다.

## Rationale

- 앱 정책 링크는 이미 노출 중이며, 법률 문서 원본의 버전 관리가 필요합니다.
- 대한민국 법령 준수 문구와 권리 고지(열람/정정/삭제/동의철회/보호의무자 권리)를
  문서에 명확히 반영해야 합니다.
- 사업자 실정보가 미확정 상태이므로, 누락 없이 입력해야 할 필수 고지 항목을
  문서 상단에 강제 노출하는 방식이 운영 리스크를 줄입니다.

## Consequences

- 장점:
  - 정책 문서 기준본이 저장소에 고정되어 배포 전 검토/추적이 쉬워집니다.
  - 법령 변경 또는 운영정책 변경 시 파일 버전 단위로 개정 이력을 관리할 수 있습니다.
- 단점:
  - 플레이스홀더를 실제 정보로 치환하지 않으면 법적 고지 의무가 미충족될 수 있습니다.

## Follow-up

1. 실제 운영자 정보(상호/주소/연락처/책임자)를 플레이스홀더에 입력
2. 법무 검토 후 정책 페이지(웹 URL)와 문구 동기화
3. 앱 상수(`legal_policy_constants.dart`)의 버전/시행일/URL을 게시본 기준으로 갱신
