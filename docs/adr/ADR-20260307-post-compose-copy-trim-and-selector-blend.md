# ADR-20260307-post-compose-copy-trim-and-selector-blend

## Status
Accepted (2026-03-07)

## Context
- Post create/edit 화면이 레퍼런스 대비 문구 밀도가 높아 보였고,
  작성 흐름에서 클론 느낌을 줄이는 요청이 있었습니다.
- 하단 `모든 사람이 댓글을 달 수 있습니다` 문구와
  `제목`/`무슨 일이 일어나고 있나요?` 플레이스홀더가
  입력 영역 집중도를 낮추는 요소로 확인되었습니다.
- 프로젝트 선택 UI는 기능적으로 필요하지만
  현재 박스형 크롬이 입력 캔버스와 분리되어 보였습니다.

## Decision
1. 하단 공개 안내 텍스트 제거:
   - 작성/수정 화면 모두 하단 툴바를 아이콘 행 중심으로 단순화.
2. 플레이스홀더 카피 정리:
   - 제목 힌트는 `(선택) 헤드라인을 입력해 주세요`로 교체.
   - 본문 힌트(`무슨 일이 일어나고 있나요?`)는 제거.
3. 프로젝트 선택 영역 톤 정렬:
   - 기존 테두리 박스형 컨테이너를 제거하고,
     라운드형 소프트 서피스 배경으로 재구성해
     타임라인형 입력 레이아웃에 자연스럽게 연결.

## Consequences
### Positive
- 입력 화면 상단-본문-하단의 시각 리듬이 단순해져
  작성 집중도가 높아집니다.
- 특정 서비스의 문구를 직접 연상시키는 느낌을 줄이고
  앱 고유 톤을 유지합니다.
- 프로젝트 선택 기능은 유지하면서도
  레이아웃 일체감이 개선됩니다.

### Trade-offs
- 본문 힌트 제거로 초기 사용자의 입력 가이드가 줄어들 수 있습니다.
- 공개 범위 안내 문구가 없어져 정책 인지가 약해질 수 있어,
  필요 시 별도 안내 지점(작성 도움말/규칙 링크) 검토가 필요합니다.
