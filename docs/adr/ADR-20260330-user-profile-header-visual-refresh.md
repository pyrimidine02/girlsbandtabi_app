# ADR-20260330: User profile header visual refresh and activity stats redesign

## Status
- Accepted (2026-03-30)

## 변경 전 문제
- 커버 이미지 위 뒤로가기 아이콘 가독성이 배경에 따라 쉽게 저하되었다.
- 상단 AppBar 타이틀(내 프로필/닉네임)과 본문 프로필 이름이 중복되어 시선이 분산됐다.
- 아바타가 슬리버 경계/레이어 순서에 따라 일부가 가려지는 케이스가 있었다.
- 활동 통계 카드가 밀집된 3열 단색 스타일로 보여 정보 우선순위와 시각적 리듬이 약했다.

## 대안
1. 기존 헤더 구조를 유지하고 색상만 조정한다.
2. 헤더 구조를 재배치해 가독성/레이어 안정성을 우선하고, 통계 UI를 카드 패널로 교체한다.

## 결정
- 대안 2를 채택한다.
- AppBar 타이틀은 제거하고, 커버 위 뒤로가기 버튼에 원형 반투명 배경을 적용한다.
- 아바타는 `SliverAppBar` 하단 전환 밴드에 배치해 커버/본문에 반씩 걸쳐 보이게 한다.
- 프로필 이름과 활성 칭호 배지는 한 줄 인라인으로 배치한다.
- 활동 통계는 3열 단색 셀에서 2열 그라데이션 카드 패널로 전면 교체한다.

## 근거
- 배경 대비가 흔들리는 이미지 기반 헤더에서는 아이콘 자체보다 배경 처리 일관성이 더 중요하다.
- 타이틀 중복 제거는 정보 계층을 단순화해 첫 스캔 속도를 높인다.
- 슬리버 경계를 넘는 오버플로우는 페인트 순서 이슈를 유발하므로,
  경계 내부 배치가 시각 안정성 측면에서 유리하다.
- 통계 카드를 2열로 키우면 라벨/값 가독성이 좋아지고, 시각적 밀집도가 줄어든다.
