# 페이지별 호출 트리거 정책표 v1.0.0

**작성일**: 2026-03-07  
**대상**: `girlsbandtabi_app` 모바일 앱  
**범위**: 프로젝트 선택 이후 페이지별 API 호출 허용/금지 정책

---

## 1) 목적

- 프로젝트 전환 시 불필요한 전역 호출/중복 호출을 차단합니다.
- 페이지가 실제로 보일 때만 해당 페이지 기능 API를 호출하도록 기준을 고정합니다.
- 서버 부하를 줄이고, 클라이언트에서 호출 의도를 명확히 관리합니다.

---

## 2) 공통 원칙 (모든 페이지 공통)

1. `selectedProject` 변경은 **컨텍스트 갱신 이벤트**로만 취급하고, 그 자체로 전역 네트워크 fan-out을 금지합니다.
2. API는 기본적으로 **현재 표시 중인 페이지(route current)** 에서만 자동 호출합니다.
3. 동일 파라미터의 동시 중복 요청은 1개만 허용합니다.
4. 커서 피드는 최초 `cursor` 없이 요청하고, 다음 페이지는 응답 `nextCursor`를 그대로 전달합니다.
5. `ads/events`는 `decisionId`가 있을 때만 전송합니다.
6. 오프스크린(IndexedStack 내 비활성 branch)에서는 주기 polling/refresh를 금지합니다.

| 홈 `/home` | 진입 시, pull-to-refresh, 홈 화면이 현재 활성 상태에서 프로젝트/유닛 변경 | `/api/v1/home/summary`, `/api/v1/ads/decision`, `/api/v1/ads/events`(decisionId 필수) | 홈 비활성 상태에서 자동 재조회 금지, 프로젝트 변경 시 즉시 `units` 선조회 금지 |
| 장소 `/places` | 진입 시, 프로젝트 변경(현재 화면일 때), 필터 변경, 지도 bounds 변경 | `/api/v1/projects/{project}/places*`, `/api/v1/projects/{project}/places/regions/*`, 필요 시 `/api/v1/projects/{project}/units` | 장소 화면이 아닌데 places API 호출 금지 |
| 라이브 `/live` | 진입 시, 프로젝트/밴드 필터 변경(현재 화면일 때) | `/api/v1/projects/{project}/live-events*`, 필요 시 `/api/v1/projects/{project}/units` | 라이브 비활성 상태에서 자동 polling 금지 |
| 게시판 피드 `/board` (추천/팔로잉/최신) | 진입 시, 탭 변경, 검색 실행/해제, 무한스크롤, 현재 화면 주기 refresh | 추천: `/api/v1/community/feed/recommended/cursor` 팔로잉: `/api/v1/community/feed/following/cursor` 최신: `/api/v1/projects/{project}/posts/cursor` | `/api/v1/community/feed/cursor`(삭제 경로) 호출 금지, `/community/subscriptions`를 비가시 상태에서 프로젝트 변경마다 호출 금지 |
| 발견 `/board/discover` | 진입 시, 무한스크롤, 현재 화면 refresh | `/api/v1/projects/{project}/posts/trending` (또는 정책 확정된 발견 전용 API) | 발견 탭에서 구독 목록(`/community/subscriptions`) 호출 금지 |
| 여행후기 `/board/travel-reviews-tab` | 진입/사용자 액션 기반 조회 | 여행후기 전용 API | 커뮤니티 추천/팔로잉/구독 API 자동 호출 금지 |
| 정보 `/info` | 뉴스 탭 표시 시, 유닛 탭 표시 시, pull-to-refresh | 뉴스: `/api/v1/projects/{project}/news*` 유닛: `/api/v1/projects/{project}/units` | 탭 비가시 상태 선조회 금지(특히 유닛 탭 미진입 시 units 호출 금지) |
| 알림 `/notifications` | 페이지 진입 시, 현재 화면 주기 refresh(40s), SSE 이벤트 수신 시 즉시 갱신, 사용자 수동 새로고침 | `/api/v1/notifications`, `/api/v1/notifications/{id}/read` | 페이지 비가시 상태에서 주기 polling 금지 |
| 검색 `/search` | 사용자 검색 입력/필터 변경 시 | 검색 API | 페이지 진입만으로 자동 대량 조회 금지 |
| 설정/프로필/기타 오버레이 | 사용자가 해당 페이지 진입 시 | 페이지별 최소 필수 API | 프로젝트 변경에 반응한 자동 재조회 금지 |

## 4) 프로젝트 선택 이벤트 전용 정책

### 허용

- `selectedProjectKey/Id` 상태 및 로컬 저장소 갱신
- 현재 표시 중 페이지가 필요 시 자신의 컨트롤러에서 `reload(forceRefresh: true)` 수행

### 금지

- 프로젝트 선택 위젯에서 직접 도메인 데이터 API 연쇄 호출
- `projectSelector`가 `projectUnitsController.load(forceRefresh: true)`를 강제 호출하는 패턴

- 프로젝트 변경 리스너에서 `_loadSubscriptions(forceRefresh: true)`를 항상 실행
- 근거:
  - `lib/features/feed/application/board_controller.dart:182-190`

## 6) 운영 기준 (적용 체크리스트)

- [ ] 프로젝트 변경 직후, 현재 route와 무관한 API 호출이 발생하지 않는다.
- [ ] `units`는 동일 프로젝트 기준 단일 요청만 발생한다.
- [ ] 추천/팔로잉 탭은 커서 규격(`nextCursor`)을 준수한다.
- [ ] 알림 polling은 `/notifications` 화면이 현재 활성일 때만 동작한다.
- [ ] 광고 이벤트는 `decisionId` 포함 시에만 발행된다.
